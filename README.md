A high-fidelity, secure, and responsive web application built for **Infant Jesus Nursing College (Kalayarkovil, Tamil Nadu)**. The platform features an interactive student-facing enrollment portal and a password-protected administrative console to collect, manage, and audit student applications in real time.

---

## 🚀 Key Features

### 🎓 Student-Facing Portal
- **Online Application Form (`#apply`)**: A focused enrollment portal capturing complete applicant profiles (academic qualifications, course preferences, contact information, and address).
- **Auto-Routing Course Pre-Selection**: Dynamic parameter parsing from course cards instantly pre-fills course selections on navigation.
- **Micro-Animations & Visuals**: Fluid card transitions and interactive elements built with `framer-motion` and `lucide-react`.
- **Intelligent Admissions Chatbot**: Interactive pre-programmed support desk with quick-replies answering course fees, hostel options, and eligibility queries.
- **Double-Submit Prevention**: Button status locks during in-flight network requests to prevent duplicate applications.

### 🛡️ Secure Administrator Dashboard (`#admin`)
- **Full-Screen Console Isolation**: Programmatic router that clears standard header/footer layout elements to focus on the workspace.
- **Authentication Safeguard**: Integrates Supabase Auth requiring secure admin credentials to access student leads.
- **Live Metrics Dashboard**: Visualizes Key Performance Indicators (KPIs) showing **Total Leads**, **New Leads**, **Called Leads**, and **Confirmed Seats**.
- **Interactive Data Table**: Features real-time text search, status filters (New, Called, Confirmed, Joined, Cancelled), and course filters.
- **Status & Counselor Notes Editor**: Interactive modal layout to modify student follow-up states and log counselor notes.
- **CSV Data Export**: Single-click browser utility to export student registries into clean Excel-compatible CSV files.

## 💾 Database Schema

The system uses a cloud-hosted PostgreSQL database. Row Level Security (RLS) is enabled to allow public/anonymous users to **insert** their applications, while blocking all unauthorized read/update operations. Only authenticated administrators can read, update, or export records.

### SQL Setup Script
Run this script inside the **SQL Editor** in your Supabase Dashboard to initialize the structure:

```sql
-- Create the applications database
CREATE TABLE IF NOT EXISTS applications (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, now()) NOT NULL,
  name TEXT NOT NULL,
  phone TEXT NOT NULL,
  email TEXT,
  gender TEXT,
  dob TEXT,
  qualification TEXT NOT NULL,
  course TEXT NOT NULL,
  address TEXT NOT NULL,
  application_id TEXT NOT NULL,
  status TEXT DEFAULT 'New' NOT NULL,
  notes TEXT
);

-- Enable Row Level Security (RLS)
ALTER TABLE applications ENABLE ROW LEVEL SECURITY;

-- Allow anonymous form inserts (for students applying)
CREATE POLICY "Enable insert for anonymous users" ON applications 
  FOR INSERT WITH CHECK (true);

-- Allow full dashboard control (for authenticated admin login)
CREATE POLICY "Enable all control for authenticated users" ON applications
  FOR ALL USING (auth.role() = 'authenticated');
```

## 💻 Local Development Setup

### Prerequisites
- Node.js (v18 or higher)
- npm or yarn

### Setup Steps
1. **Clone the repository**:
   ```bash
   git clone https://github.com/your-username/infant-nursing-website.git
   cd infant-nursing-website
   ```

2. **Install dependencies**:
   ```bash
   npm install
   ```

3. **Configure Database Keys**:
   Open [src/config.js](file:///c:/Users/aron/OneDrive/Desktop/infant/src/config.js) and enter your Supabase Project credentials:
   ```javascript
   export const SUPABASE_URL = "https://your-project-id.supabase.co";
   export const SUPABASE_ANON_KEY = "your-public-anon-key";
   ```

4. **Run development server**:
   ```bash
   npm run dev
   ```
   Open `http://localhost:5173` (or the printed port) in your web browser.

5. **Build for Production**:
   ```bash
   npm run build
   ```

## 🌐 Production Deployment (Vercel)

The easiest way to host this project for free:
1. Push your code to your GitHub account.
2. Sign in to [Vercel](https://vercel.com) using your GitHub account.
3. Import your project repository and click **Deploy**.
4. To add your custom domain (`infantjesushealth.in`), go to **Project Settings** > **Domains** in Vercel and map your DNS records (A and CNAME).
